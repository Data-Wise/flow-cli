    SPEC: v4.9.0 - Installation & Onboarding¶

    Status: Planning Created: 2026-01-05 Target: v4.9.0 Estimated Effort: 2-3 days Priority: High ‚ö°

    Overview¶

    Transform flow-cli installation from "requires ZSH plugin knowledge" to "works in 30 seconds" with zero-friction onboarding.

    Current State (v4.8.1)¶

    Homebrew (Primary): 

    brew tap data-wise/tap
brew install flow-cli
# Works immediately, but users don't know what commands exist

    Plugin Managers: 

    # Requires understanding of ZSH plugin managers
antidote install data-wise/flow-cli

    Pain Points: - ‚ùå No guided onboarding after install - ‚ùå Users don't know where to start - ‚ùå No verification that everything works - ‚ùå Missing dependencies not auto-detected - ‚ùå No first-run experience

    Target State (v4.9.0)¶

    One-Liner Install: 

    curl -fsSL https://get.flow-cli.dev/install.sh | bash
# Auto-detects: Homebrew or plugin manager
# Runs: Health check + first-run wizard
# Result: User knows exactly what to do next

    First Run: 

    $ work my-project
üëã Welcome to flow-cli! Let's get you set up.

‚úì ZSH detected
‚úì Git installed
‚ö† fzf not found (recommended for project picker)

Install recommended tools? [Y/n] y
‚úì fzf installed
‚úì bat installed

Quick tutorial? [Y/n] y
‚Üí Running: work my-project
‚Üí Log wins with: win "text"
‚Üí End session: finish

üéâ You're all set! Type 'flow help' anytime.

    Goals¶

    1. Zero-Friction Install - One command, works everywhere
    2. Auto-Discovery - Detect what's missing, offer to fix it
    3. Guided Onboarding - First-run wizard with 30-second tutorial
    4. Smart Defaults - Works out-of-box, customize later
    5. Progressive Disclosure - Show basics first, reveal advanced features gradually

    Components¶

    1. Install Script (install.sh)¶

    Location: Root of repository + hosted at get.flow-cli.dev Purpose: One-liner installation for all environments

    Features: - ‚úÖ Auto-detect OS (macOS, Linux, WSL) - ‚úÖ Auto-detect install method preference - Homebrew (if installed) ‚Üí brew install - Plugin manager (antidote, zinit, omz) ‚Üí plugin install - Manual fallback ‚Üí git clone + source - ‚úÖ Idempotent (safe to run multiple times) - ‚úÖ Version pinning support (FLOW_VERSION=v4.9.0) - ‚úÖ Dry-run mode (--dry-run) - ‚úÖ Post-install verification - ‚úÖ Trigger first-run wizard

    Architecture: 

    install.sh
‚îú‚îÄ‚îÄ detect_os()           # macOS, Linux, WSL
‚îú‚îÄ‚îÄ detect_shell()        # zsh, bash (error if not zsh)
‚îú‚îÄ‚îÄ detect_homebrew()     # Check if brew available
‚îú‚îÄ‚îÄ detect_plugin_mgr()   # antidote ‚Üí zinit ‚Üí omz ‚Üí manual
‚îú‚îÄ‚îÄ install_homebrew()    # brew tap + brew install
‚îú‚îÄ‚îÄ install_plugin()      # Plugin manager specific
‚îú‚îÄ‚îÄ install_manual()      # git clone + add to .zshrc
‚îú‚îÄ‚îÄ verify_install()      # flow --version works
‚îî‚îÄ‚îÄ run_first_setup()     # Trigger flow setup --first-run

    Error Handling: - Clear error messages with actionable fixes - Rollback on failure (remove what was added) - Support contact info on critical errors

    Testing: - Docker containers (Ubuntu 22.04, 24.04, Debian, Alpine) - macOS (Intel + Apple Silicon) - CI integration

    Implementation: 

    # In commands/doctor.zsh
_flow_doctor_fix() {
    local interactive=true
    local category="all"  # core, recommended, optional, all

    case "$1" in
        -y|--yes) interactive=false ;;
        --core) category="core" ;;
        --recommended) category="recommended" ;;
        --optional) category="optional" ;;
    esac

    _doctor_check_and_install "core" "$interactive"
    [[ "$category" != "core" ]] && _doctor_check_and_install "recommended" "$interactive"
    [[ "$category" == "all" || "$category" == "optional" ]] && _doctor_check_and_install "optional" "$interactive"
}

    Implementation: 

    # commands/setup.zsh
flow_setup() {
    local mode="interactive"  # interactive, quick, unattended
    local force=false

    # Check if already setup
    if [[ -f ~/.config/flow-cli/.setup_complete ]] && ! $force; then
        echo "‚úì Already setup. Use --force to run again."
        return 0
    fi

    case "$1" in
        --first-run) mode="interactive" ;;
        --quick) mode="quick" ;;
        --unattended) mode="unattended" ;;
        --force) force=true ;;
    esac

    _setup_welcome
    _setup_verify_install
    _setup_install_tools "$mode"
    _setup_configure_projects "$mode"
    _setup_tutorial "$mode"
    _setup_complete
}

    4. Auto-Discovery & Smart Defaults¶

    Project Root Detection: 

    # Try in order:
1. $FLOW_PROJECTS_ROOT (if set)
2. ~/projects (if exists)
3. ~/dev (if exists)
4. ~/code (if exists)
5. ~ (fallback, show all subdirectories)

    Shell Integration: 

    # Auto-detect shell config file
1. ~/.zshrc (most common)
2. ~/.config/zsh/.zshrc (XDG)
3. $ZDOTDIR/.zshrc (if ZDOTDIR set)

    Package Manager Detection: 

    # Order of preference
1. brew (macOS/Linux)
2. apt-get (Debian/Ubuntu)
3. pacman (Arch)
4. dnf (Fedora)
5. yum (RHEL/CentOS)

    Risks & Mitigations¶

    Risk Impact Mitigation
    Install script breaks on new OS High Extensive Docker testing, CI matrix
    Homebrew tap conflicts Medium Use unique tap name, test with other taps
    Plugin manager conflicts Medium Detect existing installs, offer choices
    Network failures during install High Add retry logic, offline fallback
    Wizard too long Low Add --quick mode, make skippable
