    Master Dispatcher Guide¶

    Purpose: Complete reference for all 12 flow-cli dispatchers Audience: All users (beginner ‚Üí intermediate ‚Üí advanced) Format: Progressive disclosure (basics ‚Üí advanced features) Version: v5.17.0-dev Last Updated: 2026-01-24

    Overview¶

    flow-cli uses dispatchers - single-letter or short commands that group related functionality. Each dispatcher follows a consistent pattern:

    <dispatcher> <subcommand> [options] [args]

    Example: 

    g status              # Git status
r test                # Run R tests
teach init            # Initialize course

    The 12 Dispatchers¶

    Dispatcher Domain Commands Complexity
    g Git workflows 20+ Beginner ‚Üí Advanced
    cc Claude Code 4 Beginner
    r R packages 10+ Intermediate
    qu Quarto publishing 8+ Intermediate
    mcp MCP servers 8 Intermediate
    obs Obsidian notes 6 Beginner
    wt Worktrees 6 Advanced
    dot Dotfiles & secrets 12+ Intermediate ‚Üí Advanced
    teach Teaching workflow 15+ Intermediate ‚Üí Advanced
    tm Terminal manager 5 Beginner
    prompt Prompt engine 3 Beginner
    v Vibe coding mode 4 Beginner

    How to Use This Guide¶

    For Beginners¶

    Start with the Basics section of each dispatcher: - Core commands you'll use daily - Simple examples - Common use cases

    Recommended Learning Order: 1. tm - Terminal (easiest) 2. cc - Claude Code 3. g basics - Git 4. obs - Obsidian (if you use it)

    For Intermediate Users¶

    Explore Intermediate sections: - Advanced subcommands - Workflow patterns - Integration with other tools

    Recommended Path: 1. g feature workflow 2. r or qu (based on your work) 3. teach (if teaching) 4. dot - Secret management

    For Advanced Users¶

    Deep dive into Advanced sections: - Power user features - Automation patterns - Edge cases and customization

    Focus Areas: 1. wt - Parallel development with worktrees 2. dot advanced - Secret rotation, automation 3. teach advanced - AI integration 4. mcp - Server management

    Dispatchers¶

    g Dispatcher¶

    Domain: Git workflows Complexity: Beginner ‚Üí Advanced Most Used: Yes (daily git operations)

    Basics (Beginner)¶

    What it does: Simplifies common git operations with short commands.

    Essential Commands¶

    Check status: 

    g status
g st                 # Alias for status

    Stage changes: 

    g add .              # Add all changes
g add file.txt       # Add specific file

    Commit: 

    g commit "feat: add user authentication"
g cm "fix: resolve login bug"

    Push/Pull: 

    g push               # Push to remote
g pull               # Pull from remote

    View history: 

    g log                # Full log
g log --oneline      # Compact log
g log --graph        # Visual graph

    Show changes: 

    g diff               # Unstaged changes
g diff --staged      # Staged changes

    Expected Workflow (Beginner): 

    # 1. Check what changed
g status

# 2. Stage changes
g add .

# 3. Commit
g commit "feat: add feature X"

# 4. Push
g push

    Intermediate¶

    Feature Branch Workflow¶

    Create feature branch: 

    g feature start my-feature

    This: - Creates feature/my-feature from dev (or main) - Switches to new branch - Ready to work

    Push feature: 

    g feature push

    Create PR: 

    g feature pr
# Uses gh cli to create PR to dev branch

    Finish feature (after merge): 

    g feature finish

    This: - Switches back to dev - Pulls latest changes - Deletes feature branch locally

    Complete Feature Workflow: 

    # Day 1: Start
g feature start user-profiles
# ... work on feature ...
g add .
g commit "feat: add user profile page"
g feature push

# Day 2: Create PR
g feature pr

# Day 5: PR merged
g feature finish

    Branch Management¶

    List branches: 

    g branch              # Local branches
g branch -r           # Remote branches
g branch -a           # All branches

    Create branch: 

    g checkout -b new-branch
g co -b new-branch    # Short form

    Switch branches: 

    g checkout branch-name
g co branch-name

    Delete branch: 

    g branch -d branch-name        # Safe delete (merged only)
g branch -D branch-name        # Force delete

    Cleanup merged branches: 

    g feature prune
# Deletes all local branches that are merged into dev

    Stash Commands¶

    Save work in progress: 

    g stash
g stash save "WIP: refactoring auth"

    List stashes: 

    g stash list

    Apply stash: 

    g stash pop            # Apply and remove
g stash apply          # Apply but keep

    Clear all stashes: 

    g stash clear

    Advanced¶

    Rebase & History Rewriting¶

    Rebase onto branch: 

    g rebase dev
g rebase origin/dev

    Interactive rebase: 

    g rebase -i HEAD~3
# Allows: pick, reword, edit, squash, fixup, drop

    Squash last 3 commits: 

    g rebase -i HEAD~3
# Change 'pick' to 'squash' for commits to merge

    Cherry-pick¶

    Pick specific commit: 

    g cherry-pick <commit-hash>

    Pick range: 

    g cherry-pick commit1..commit2

    Reset & Restore¶

    Undo last commit (keep changes): 

    g reset HEAD~1

    Undo last commit (discard changes): 

    g reset --hard HEAD~1

    Restore file from specific commit: 

    g restore --source=<commit> file.txt

    Restore staged file: 

    g restore --staged file.txt

    Sync Workflow¶

    Update branch with latest: 

    g sync

    This: - Fetches from origin - Rebases current branch onto origin/dev - Pushes if needed

    Force push (DANGEROUS): 

    g push --force-with-lease    # Safer force push

    Reference¶

    Complete g Dispatcher Command List **Basic Commands:** - `g status` / `g st` - Show working tree status - `g add ` - Stage changes - `g commit ` / `g cm ` - Create commit - `g push` - Push to remote - `g pull` - Pull from remote - `g diff` - Show changes - `g log` - Show commit history **Branch Commands:** - `g branch` - List/create branches - `g checkout ` / `g co ` - Switch branches - `g checkout -b ` - Create and switch to new branch - `g branch -d ` - Delete branch - `g feature start ` - Start feature branch - `g feature push` - Push current feature - `g feature pr` - Create PR - `g feature finish` - Finish and cleanup - `g feature prune` - Delete merged branches - `g feature list` - List all features **Stash Commands:** - `g stash` - Stash changes - `g stash list` - List stashes - `g stash pop` - Apply and remove stash - `g stash apply` - Apply stash - `g stash drop` - Remove stash - `g stash clear` - Clear all stashes **History Commands:** - `g log` - Full log - `g log --oneline` - Compact log - `g log --graph` - Visual graph - `g log --all` - All branches - `g rebase ` - Rebase onto branch - `g rebase -i ` - Interactive rebase - `g cherry-pick ` - Cherry-pick commit - `g reset ` - Reset to commit - `g reset --hard ` - Hard reset **Advanced Commands:** - `g sync` - Fetch, rebase, push - `g restore ` - Restore file - `g restore --staged ` - Unstage file

    cc Dispatcher¶

    Domain: Claude Code launcher Complexity: Beginner Most Used: Yes (if using Claude Code)

    Basics (Beginner)¶

    What it does: Launches Claude Code CLI with smart defaults.

    Essential Commands¶

    Launch in current directory: 

    cc

    Output: [Launches Claude Code CLI in current directory]

    Launch with project picker: 

    cc pick

    This: - Shows interactive project list (fzf) - Select project - Launch Claude Code in that directory

    Launch in yolo mode (auto-approve all): 

    cc yolo

    WARNING: Auto-approves all permission requests. Only use in trusted environments.

    Show help: 

    cc help

    Intermediate¶

    Integration with work Command¶

    # Start work session, then launch Claude
work my-project
cc
# Claude launches in my-project directory

    Custom Launch Options¶

    Environment variables respected: 

    export CLAUDE_CODE_MODEL="opus"    # Use Opus model
export CLAUDE_CODE_YOLO=1          # Auto-approve by default

    Reference¶

    Complete cc Dispatcher Command List - `cc` - Launch Claude Code in current directory - `cc pick` - Interactive project picker, then launch - `cc yolo` - Launch with auto-approval - `cc help` - Show help

    r Dispatcher¶

    Domain: R package development Complexity: Intermediate Most Used: Yes (if doing R development)

    Basics (Beginner)¶

    What it does: Streamlines R package development workflow.

    Essential Commands¶

    Run tests: 

    r test

    Output:

    Build documentation: 

    r doc

    Check package: 

    r check

    Runs R CMD check - comprehensive package validation.

    Install package: 

    r install

    Installs package locally for testing.

    Intermediate¶

    Development Workflow¶

    Load package in R session: 

    r load

    Runs devtools::load_all() - fast iteration without full install.

    Build source tarball: 

    r build

    Creates .tar.gz file for distribution.

    Run example: 

    r example my_function

    Runs examples from function documentation.

    Complete Development Cycle¶

    # 1. Write function with roxygen comments
# 2. Generate documentation
r doc

# 3. Run tests
r test

# 4. Check package
r check

# 5. Install locally
r install

# 6. If all pass, commit
g commit "feat: add new function"

    Advanced¶

    Custom Test Options¶

    Run specific test file: 

    r test test-my-function.R

    Run with coverage: 

    r coverage

    Shows test coverage report.

    Run with profiling: 

    r profile

    Profiles package performance.

    Reference¶

    Complete r Dispatcher Command List **Basic:** - `r test` - Run testthat tests - `r doc` - Build documentation (roxygen2) - `r check` - R CMD check - `r install` - Install package - `r build` - Build source tarball - `r load` - Load with devtools::load_all() **Advanced:** - `r test ` - Run specific test - `r coverage` - Test coverage report - `r example ` - Run function example - `r profile` - Profile performance - `r help` - Show help

    qu Dispatcher¶

    Domain: Quarto publishing Complexity: Intermediate Most Used: Yes (if using Quarto)

    Basics (Beginner)¶

    What it does: Simplifies Quarto document rendering and publishing.

    Essential Commands¶

    Preview document: 

    qu preview
qu preview document.qmd

    Starts preview server at http://localhost:4567/

    Render document: 

    qu render
qu render document.qmd

    Renders to default output format (usually HTML).

    Render to PDF: 

    qu render document.qmd --to pdf

    Render to Word: 

    qu render document.qmd --to docx

    Intermediate¶

    Website/Book Publishing¶

    Render entire website: 

    qu render --website

    Publish to GitHub Pages: 

    qu publish gh-pages

    Create new project: 

    qu create website
qu create book
qu create manuscript

    Reference¶

    Complete qu Dispatcher Command List - `qu preview [file]` - Start preview server - `qu render [file]` - Render document - `qu render --to ` - Render to specific format - `qu render --website` - Render entire website - `qu publish gh-pages` - Publish to GitHub Pages - `qu create ` - Create new project - `qu help` - Show help

    mcp Dispatcher¶

    Domain: MCP server management Complexity: Intermediate Most Used: Yes (if using MCP servers)

    Basics (Beginner)¶

    What it does: Manages MCP (Model Context Protocol) servers.

    Essential Commands¶

    List servers: 

    mcp list
mcp ls

    Output: 

    statistical-research (running)
rforge (running)
nexus (running)
playwright (stopped)

    Show status: 

    mcp status

    Shows detailed status table.

    Start server: 

    mcp start statistical-research

    Stop server: 

    mcp stop statistical-research

    Restart server: 

    mcp restart statistical-research

    Intermediate¶

    Show logs: 

    mcp logs statistical-research

    Tails server logs in real-time.

    Test server: 

    mcp test statistical-research

    Tests server connectivity and health.

    Reference¶

    Complete mcp Dispatcher Command List - `mcp list` / `mcp ls` - List all servers - `mcp status` - Show detailed status - `mcp start ` - Start server - `mcp stop ` - Stop server - `mcp restart ` - Restart server - `mcp logs ` - Show logs - `mcp test ` - Test connectivity - `mcp help` - Show help
