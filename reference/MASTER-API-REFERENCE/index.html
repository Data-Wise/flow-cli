    Logging & Output¶

    _flow_log_success¶

    Logs success message with green checkmark.

    Signature: 

    _flow_log_success "message"

    Parameters: - $1 - Message to log

    Returns: - Always returns 0

    Example: 

    _flow_log_success "Project initialized successfully"
# Output: ‚úÖ Project initialized successfully

    _flow_log_error¶

    Logs error message with red X.

    Signature: 

    _flow_log_error "message"

    Parameters: - $1 - Error message

    Returns: - Always returns 1

    Example: 

    _flow_log_error "Configuration file not found"
# Output: ‚ùå Configuration file not found

    _flow_log_warning¶

    Logs warning message with yellow warning sign.

    Signature: 

    _flow_log_warning "message"

    Parameters: - $1 - Warning message

    Project Utilities¶

    _flow_find_project_root¶

    Finds git repository root from current directory.

    Signature: 

    _flow_find_project_root

    Parameters: - None (uses current directory)

    Returns: - 0 - Success, prints root path to stdout - 1 - Not in git repository

    Example: 

    root=$(_flow_find_project_root)
if [[ $? -eq 0 ]]; then
    echo "Project root: $root"
else
    echo "Not in git repository"
fi

    _flow_detect_project_type¶

    Detects project type from directory structure.

    Signature: 

    _flow_detect_project_type "/path/to/project"

    Parameters: - $1 - Project directory path

    Returns: - 0 - Success, prints project type to stdout - 1 - Unknown project type

    Supported Types: - node - Node.js (package.json) - r - R package (DESCRIPTION with Package:) - python - Python (pyproject.toml, setup.py) - quarto - Quarto (_quarto.yml) - teaching - Teaching course (course-config.yml) - mcp - MCP server (mcp-server/ directory)

    Example: 

    type=$(_flow_detect_project_type "$PWD")
echo "Project type: $type"
# Output: Project type: node

    Secret Storage¶

    _flow_keychain_set¶

    Stores secret in macOS Keychain with Touch ID.

    Signature: 

    _flow_keychain_set "SECRET_NAME" "secret_value"

    Parameters: - $1 - Secret name (e.g., GITHUB_TOKEN) - $2 - Secret value

    Returns: - 0 - Success - 1 - Failed to store (keychain locked, permission denied)

    Security: - Uses macOS Keychain - Requires Touch ID (or password fallback) - Encrypted at rest

    Example: 

    if _flow_keychain_set "GITHUB_TOKEN" "ghp_xxxxxxxxxxxx"; then
    _flow_log_success "Token stored in keychain"
else
    _flow_log_error "Failed to store token"
fi

    _flow_keychain_get¶

    Retrieves secret from macOS Keychain.

    Signature: 

    _flow_keychain_get "SECRET_NAME"

    Parameters: - $1 - Secret name

    Returns: - 0 - Success, prints value to stdout - 1 - Secret not found or keychain locked

    Example: 

    token=$(_flow_keychain_get "GITHUB_TOKEN")
if [[ $? -eq 0 ]]; then
    export GITHUB_TOKEN="$token"
else
    _flow_log_error "Token not found in keychain"
fi

    _flow_keychain_delete¶

    Deletes secret from keychain.

    Signature: 

    _flow_keychain_delete "SECRET_NAME"

    Parameters: - $1 - Secret name

    Returns: - 0 - Success - 1 - Failed to delete

    Example: 

    _flow_keychain_delete "OLD_TOKEN"

    _flow_keychain_list¶

    Lists all flow-cli secrets in keychain.

    _flow_keychain_list

    Example:

    Token Validation¶

    _flow_git_validate_token¶

    Validates GitHub token before remote operations.

    Signature: 

    _flow_git_validate_token

    Parameters: - None (uses GITHUB_TOKEN from keychain or environment)

    Returns: - 0 - Token valid - 1 - Token invalid or expired

    Caching: - Uses 5-minute cache (v5.17.0+) - Cache file: ~/.cache/flow/doctor/tokens.cache

    Example: 

    if _flow_git_validate_token; then
    git push origin dev
else
    _flow_log_error "Invalid token, run: dot secret rotate GITHUB_TOKEN"
    return 1
fi

    Concept Extraction¶

    _teach_extract_concepts¶

    Extracts learning concepts from Quarto frontmatter.

    Signature: 

    _teach_extract_concepts "file.qmd"

    Parameters: - $1 - Quarto file path

    Returns: - 0 - Success, outputs JSON to stdout - 1 - Failed to parse

    Output Format: 

    {
  "concepts": [
    {
      "id": "linear-regression",
      "name": "Linear Regression",
      "bloom": "Understand",
      "complexity": "Medium",
      "prerequisites": ["basic-statistics"]
    }
  ]
}

    Example: 

    concepts=$(_teach_extract_concepts "lectures/week-01/regression.qmd")
echo "$concepts" | jq '.concepts[] | .name'

    Analysis Cache¶

    _teach_cache_get¶

    Retrieves cached analysis result.

    Signature: 

    _teach_cache_get "file.qmd"

    Parameters: - $1 - File path

    Returns: - 0 - Cache hit, outputs cached data - 1 - Cache miss

    Caching Strategy: - SHA-256 hash of file content - flock for concurrent access - 24-hour TTL

    Example: 

    if cached=$(_teach_cache_get "lecture.qmd"); then
    echo "Using cached analysis"
    echo "$cached"
else
    echo "Cache miss, analyzing..."
    result=$(_teach_analyze_file "lecture.qmd")
    _teach_cache_set "lecture.qmd" "$result"
fi

    Documentation Standards¶

    Function naming: - Private: _flow_* (underscore prefix) - Public: flow_* (no underscore) - Dispatcher: <dispatcher>_* (e.g., g_feature_start)

    Parameter documentation: - Always document all parameters - Mark optional parameters - Provide examples

    Return values: - Always document return codes - 0 = success, non-zero = error - Print output to stdout, errors to stderr
