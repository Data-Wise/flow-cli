## üí° The "vibe" Pattern - Why It's Genius

### What Makes "vibe" Perfect

1. ‚úÖ **Evocative** - Captures feeling/flow state
2. ‚úÖ **Short** - Easy to type (4 letters, 1 syllable)
3. ‚úÖ **Context-aware** - Adapts to situation
4. ‚úÖ **Memorable** - Natural word, not arbitrary
5. ‚úÖ **Expandable** - Can add subcommands
6. ‚úÖ **Fun** - Less robotic, more human

### The "vibe" Philosophy

- **"What's the vibe?"** = What should I work on?
- **"Get in the vibe"** = Enter flow state
- **"Check the vibe"** = See status/mood
- **"Vibe test"** = Run tests in context
- **"Vibe with it"** = Go with the flow

This is **ADHD-friendly** because:

- Low cognitive load
- Taps into emotion/mood
- Context-aware (reduces decisions)
- Natural language

**Pros:**

### Option 3: Hybrid (Recommended ‚≠ê)

**High-level workflow commands stay top-level:**

```bash
vibe test           # Workflow automation
work medfit         # Start session
focus 25            # Time management
win "fixed bug"     # Achievement logging
```

**Utility verbs at top level (frequently used):**

```bash
pick                # Most frequent: context-aware selection
pick file           # Explicit selection
pick test
pick status

switch              # Git: frequently used
stage               # Git: frequently used
review              # Git: code review
```

**Less frequent verbs under vibe:**

```bash
vibe browse         # Git: browse commits (less frequent)
vibe hunt           # Find in history (less frequent)
vibe trace          # Follow execution (less frequent)
```

**Why This Works:**

- ‚úÖ Frequently used = short commands (pick, switch, stage)
- ‚úÖ Less frequent = organized under vibe
- ‚úÖ High-level workflows = top level (vibe, work, focus)
- ‚úÖ Best of both worlds
