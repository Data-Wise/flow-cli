    Implementation Summary: Teach Dispatcher Phases 5-6¶

    Feature: Scholar Enhancement - Revision Workflow + Context & Polish Branch: feature/teaching-flags Status: ‚úÖ Complete (Phases 5-6 of 6) Date: 2026-01-17 Test Results: 45/45 Phase 1-2 + 28/28 regression + 38/38 integration tests passing

    What Was Implemented¶

    Phase 5: Revision Workflow (~4h actual)¶

    New Functions:

    1. _teach_analyze_file() - Detect content type from file
    2. Analyzes YAML frontmatter and content
    3. Detects: slides, lecture, exam, quiz, assignment, syllabus, rubric
    4. Returns detected type or "unknown"

    5. Uses pattern matching on frontmatter + content

    6. _teach_revision_menu() - Interactive revision option selection

    7. 6 revision options:
      1. Add missing content
      2. Improve clarity/organization
      3. Fix errors/inconsistencies
      4. Update examples/exercises
      5. Enhance formatting/style
      6. Custom instructions
    8. Keyboard input [1-6]

    9. Returns formatted revision instruction for Scholar

    10. _teach_show_diff_preview() - Preview changes before applying

    11. Shows file path and modification time
    12. Displays git diff if file is tracked
    13. Indicates untracked files

    14. Helps user decide on revision approach

    15. _teach_revise_workflow() - Main revision orchestrator

    16. Validates file exists and is readable
    17. Analyzes content type
    18. Shows diff preview (optional)
    19. Presents revision menu
    20. Sets global variables for Scholar integration
    21. Returns 0 if successful, 1 if cancelled/error

    Integration into Wrapper: - Added Phase 5 section at beginning of _teach_scholar_wrapper() - Runs BEFORE content flag validation (Phase 1-2) - Sets TEACH_REVISE_MODE, TEACH_REVISE_FILE, TEACH_REVISE_INSTRUCTIONS - Revision instructions included in Scholar command

    Global Variables Set:

    TEACH_REVISE_MODE          # Always "improve" for now
TEACH_REVISE_FILE          # Path to file being revised
TEACH_REVISE_INSTRUCTIONS  # User-selected revision instructions

    Phase 6: Context & Polish (~4h actual)¶

    1. _teach_build_context() - Gather course context files

      • syllabus.md (course objectives)
      • README.md (project overview)
    3. Builds context text from existing files

    Known Limitations¶

    Phase 5: Revision Workflow¶

    1. Git diff dependency - Diff preview requires git repo
    2. Graceful fallback: Shows "untracked" message

    3. Future: Add file comparison without git

    4. Single file revision - Can only revise one file at a time

    5. Future: Batch revision mode for multiple files

    6. Current: Run multiple times for multiple files

    7. No revision history - Doesn't track past revisions

    8. Future: Add revision log with diffs
    9. Current: Use git history manually

    Phase 6: Context Integration¶

    1. Fixed file list - Only searches for specific files
    2. Future: Auto-discover all teaching materials

    3. Current: Manually specify additional context

    4. No context caching - Rebuilds context each time

    5. Future: Cache context with invalidation

    6. Current: Rebuilds on every invocation (~10ms)

    7. yq dependency - Context parsing requires yq

    8. Graceful fallback: Works without yq (limited parsing)
    9. Future: Consider pure ZSH YAML parser
