    Git Hook System Implementation¶

    Status: ‚úÖ Complete Version: v1.0.0 Date: 2026-01-20

    Overview¶

    Complete implementation of the 5-layer pre-commit hook system for flow-cli Quarto workflow as specified in IMPLEMENTATION-INSTRUCTIONS.md (Week 1-2).

    Files Created¶

    Hook Templates (3 files)¶

    1. lib/hooks/pre-commit-template.zsh (484 lines)

    3. Interactive error handling
    4. Parallel rendering support

    5. Timing data for commit messages

    6. lib/hooks/pre-push-template.zsh (235 lines)

    7. Production branch validation
    8. Site build verification

    9. _freeze/ detection for pushes

    10. lib/hooks/prepare-commit-msg-template.zsh (64 lines)

    Features Implemented¶

    Pre-Commit Hook (5 Layers)¶

    Layer 1: YAML Frontmatter Validation¶

    • ‚úÖ Check for --- delimiters
    • ‚úÖ Parse YAML with yq
    • ‚úÖ Validate syntax
    • ‚úÖ Error reporting with file location

    Layer 2: Quarto Syntax Check¶

    • ‚úÖ quarto inspect validation
    • ‚úÖ Graceful degradation if quarto not installed
    • ‚úÖ Error output capture

    Layer 3: Full Render (Optional)¶

    • ‚úÖ Controlled by QUARTO_PRE_COMMIT_RENDER=1
    • ‚úÖ Parallel rendering for multiple files
    • ‚úÖ Configurable max parallel jobs (QUARTO_MAX_PARALLEL)
    • ‚úÖ Background job management
    • ‚úÖ Failure detection and reporting

    Layer 4: Empty Code Chunk Detection (Warning)¶

    • ‚úÖ Detect empty R chunks: ```{r} ```
    • ‚úÖ Detect empty Python chunks: ```{python} ```
    • ‚úÖ Non-blocking warnings with line numbers
    • ‚úÖ Continue commit despite warnings (with prompt)

    Layer 5: Image Reference Validation (Warning)¶

    • ‚úÖ Extract markdown images: ![alt](path)
    • ‚úÖ Extract knitr images: include_graphics("path")
    • ‚úÖ Check file existence (relative paths)
    • ‚úÖ Skip URL validation (http/https)
    • ‚úÖ Non-blocking warnings

    Special: _freeze/ Commit Prevention¶

    • ‚úÖ Hard block on _freeze/ in staged files
    • ‚úÖ Helpful error message
    • ‚úÖ Suggest unstaging command

    Interactive Error Handling¶

    • ‚úÖ "Commit anyway? [y/N]" prompts
    • ‚úÖ Timeout after 30 seconds (defaults to No)
    • ‚úÖ Separate prompts for errors vs warnings
    • ‚úÖ Clear messaging about validation status

    Parallel Rendering¶

    • ‚úÖ Process multiple .qmd files concurrently
    • ‚úÖ Max parallel jobs configurable (default: 4)
    • ‚úÖ Background job tracking with PIDs
    • ‚úÖ Wait for all jobs to complete
    • ‚úÖ Collect failures across parallel runs
    • ‚úÖ Performance timing

    Pre-Push Hook¶

    Production Branch Validation¶

    • ‚úÖ Detect protected branches (main, production, gh-pages)
    • ‚úÖ Require _site/ directory exists
    • ‚úÖ Verify _site/index.html exists
    • ‚úÖ Check site has minimum files (>= 3)
    • ‚úÖ Check site freshness (< 24 hours)
    • ‚úÖ Block _freeze/ in commits
    • ‚úÖ Lenient mode for draft/feature/dev branches

    Prepare-Commit-Msg Hook¶

    • ‚úÖ Append validation timing: [validation: Xs]
    • ‚úÖ Optional summary: [validation: Xs] N files validated: X errors, Y warnings
    • ‚úÖ Controlled by QUARTO_COMMIT_TIMING=1
    • ‚úÖ Controlled by QUARTO_COMMIT_SUMMARY=1
    • ‚úÖ Read from temporary files created by pre-commit

    Version Management¶

    • ‚úÖ Semantic versioning (X.Y.Z)
    • ‚úÖ Version embedded in hook files
    • ‚úÖ Compare versions (equal, greater, lesser)
    • ‚úÖ Detect outdated hooks
    • ‚úÖ Upgrade command with preview
    • ‚úÖ Downgrade protection (with --force override)

    Hook Installation¶

    • ‚úÖ Install all hooks: teach hooks install
    • ‚úÖ Upgrade hooks: teach hooks upgrade
    • ‚úÖ Check status: teach hooks status
    • ‚úÖ Uninstall hooks: teach hooks uninstall
    • ‚úÖ Force reinstall: teach hooks install --force
    • ‚úÖ Backup non-flow hooks before overwrite
    • ‚úÖ Verify Quarto project before install
    • ‚úÖ Verify git repository exists
    • ‚úÖ Make hooks executable automatically

    Configuration Options¶

    All hooks respect environment variables:

    # Enable full rendering on commit (default: off)
export QUARTO_PRE_COMMIT_RENDER=1

# Enable parallel rendering (default: on)
export QUARTO_PARALLEL_RENDER=1

# Max parallel jobs (default: 4)
export QUARTO_MAX_PARALLEL=8

# Add timing to commit messages (default: on)
export QUARTO_COMMIT_TIMING=1

# Add validation summary to commits (default: off)
export QUARTO_COMMIT_SUMMARY=1

    Usage Examples¶

    Installation¶

    # Navigate to Quarto project
cd ~/teaching/stat-440

# Install hooks (first time)
teach hooks install

# Check hook status
teach hooks status

# Upgrade hooks (when flow-cli updates)
teach hooks upgrade

    Daily Workflow¶

    # Edit a .qmd file
vim lectures/week-01.qmd

# Stage changes
git add lectures/week-01.qmd

# Commit (hooks run automatically)
git commit -m "feat: add week 1 lecture"
# ‚Üí Pre-commit hook validates:
#   ‚úì YAML frontmatter
#   ‚úì Quarto syntax
#   ‚ö† Empty code chunk detected (line 45)
#   Continue commit? [y/N] y

# Push to remote
git push origin main
# ‚Üí Pre-push hook validates:
#   ‚úì _site/ directory exists
#   ‚úì _site/index.html exists
#   ‚úì Site has 127 files
#   ‚úì Site built 2 hours ago

    Enable Full Rendering¶

    # Add to ~/.zshrc for persistence
export QUARTO_PRE_COMMIT_RENDER=1

# Or set per-commit
QUARTO_PRE_COMMIT_RENDER=1 git commit -m "..."

    Parallel Rendering¶

    # Increase parallel jobs for faster validation
export QUARTO_MAX_PARALLEL=8

# Commit multiple files
git add lectures/*.qmd
git commit -m "feat: add all week 1 lectures"
# ‚Üí Renders 5 files in parallel (max 8 jobs)

    Version Management¶

    Hooks use semantic versioning (X.Y.Z) embedded in the file:

    # Version: 1.0.0

    Version comparison logic: - 1.0.0 == 1.0.0 ‚Üí Up to date - 1.1.0 > 1.0.0 ‚Üí Upgrade available - 2.0.0 > 1.5.0 ‚Üí Newer version installed

    Integration with flow-cli¶

    Commands Added¶

    These commands will be added to the teach dispatcher:

    teach hooks install      # Install all hooks
teach hooks upgrade      # Upgrade to latest version
teach hooks status       # Show hook status
teach hooks uninstall    # Remove flow-managed hooks

    teach-dispatcher Integration¶

    The hook installer will be sourced in the teach dispatcher:

    # Source hook installer (v5.15.0+)
if [[ -z "$_FLOW_HOOK_INSTALLER_LOADED" ]]; then
    local hook_installer_path="${0:A:h:h}/hook-installer.zsh"
    [[ -f "$hook_installer_path" ]] && source "$hook_installer_path"
    typeset -g _FLOW_HOOK_INSTALLER_LOADED=1
fi

    Then add hook commands to the dispatcher case statement:

    teach() {
  case "$1" in
    hooks)
      shift
      case "$1" in
        install)   shift; _install_git_hooks "$@" ;;
        upgrade)   shift; _upgrade_git_hooks "$@" ;;
        status)    shift; _check_all_hooks "$@" ;;
        uninstall) shift; _uninstall_git_hooks "$@" ;;
        *)         _teach_hooks_help ;;
      esac
      ;;
    # ... other commands
  esac
}

    • 5 files sequential: ~12.5-27.5s (5 √ó 2.5-5.5s)
    • 5 files parallel (4 jobs): ~3.5-6.5s
    • Speedup: 3.5-4.2x

    Troubleshooting¶

    Common Issues¶

    1. Hooks not running¶

    # Check if hooks are installed
teach hooks status

# Reinstall if needed
teach hooks install --force

    2. "yq not found" warning¶

    # Install yq (macOS)
brew install yq

# Or skip YAML validation (not recommended)
# Hook will continue with warnings

    3. "quarto not found" warning¶

    # Install Quarto
brew install quarto

# Or the hook will skip syntax validation

    4. Commit blocked by _freeze/¶

    # Unstage _freeze/ directory
git restore --staged _freeze/

# Add to .gitignore if needed
echo "_freeze/" >> .gitignore

    5. Pre-push blocked: "_site/ not found"¶

    # Render the site before pushing
quarto render

# Then push again
git push

    Debug Mode¶

    Enable debug output by setting FLOW_DEBUG=1:

    FLOW_DEBUG=1 git commit -m "..."
