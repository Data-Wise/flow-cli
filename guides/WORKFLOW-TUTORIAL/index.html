    3Ô∏è‚É£ work - Start Working¶

    üéØ What it does¶

    Opens your editor in a project directory.

    üíª How to use it¶

    # Full project name
work mediationverse

# Short name (fuzzy match)
work med            # ‚Üí mediationverse
work stat           # ‚Üí stat-440

# From current directory
cd ~/projects/r-packages/active/mediationverse
work .              # Open current project

# No args (not implemented yet)
work                # Would open current project or show picker

    ‚úÖ When to use it¶

    • After just-start: Pick a project ‚Üí start working
    • After dash: See all ‚Üí pick one ‚Üí start
    • Direct jump: You know what you want to work on
    • Resume work: Jump back to a project

    üé® What you'll see¶

    work mediationverse

    What happens:

    1. Changes to project directory
    2. Detects project type (R package, Quarto, etc.)
    3. Opens appropriate editor:
    4. R package ‚Üí RStudio or Emacs
    5. Quarto ‚Üí VS Code or Cursor
    6. Research ‚Üí Emacs
    7. Dev tools ‚Üí VS Code

    Editor opens automatically!

    4Ô∏è‚É£ status - Update Project Status¶

    üéØ What it does¶

    Updates a project's status, priority, task, and progress.

    üíª How to use it¶

    Quick mode (if you know all values):

    Create mode (start tracking a new project):

    Show mode (just view, don't update):

    ‚úÖ When to use it¶

    • End of work session: Update progress before stopping
    • Status change: Mark as paused, blocked, or active
    • Priority change: Adjust priority as things change
    • Task complete: Update to next task
    • New project: Create .STATUS file

    Fields explained:

    • Status: What stage is the project in?
    • active = Currently working on
    • ready = Ready to start
    • paused = On hold

    • blocked = Waiting for something

    • Priority: How urgent?

    • P1 = Important (this week)

    • P2 = Normal (when you can)

    • Task: What's the next action?

    • Be specific: "Run simulation" not "Work on it"

    • Progress: How far along? (0-100%)

    ‚ùì Common Questions¶

    Q: Where is my project data stored? A: In each project's .STATUS file.

    Q: What if I don't have a .STATUS file? A: Use status <project> --create to create one.

    Q: Can I edit .STATUS manually? A: Yes, but use status command - it's easier!

    Q: What if I have many projects? A: Use category filters: dash teaching

    Q: How does just-start pick? A: Priority order: P0 active ‚Üí P1 active ‚Üí any active ‚Üí most recent

    Q: Can I use this with my existing workflow? A: Yes! It works with work, r, cc, qu, etc.
