    Comprehensive Help System Guide¶

    Version: v5.14.0 Last Updated: 2026-01-20 Status: ‚úÖ Production Ready

    Overview¶

    The comprehensive help system provides 18 detailed help functions for all teach dispatcher commands, following consistent formatting, progressive disclosure patterns, and ADHD-friendly design principles.

    What's Included¶

    • ‚úÖ 18 Help Functions - Complete coverage of all teach commands
    • ‚úÖ Consistent Formatting - Same structure across all help text
    • ‚úÖ Progressive Disclosure - Quick Start ‚Üí Options ‚Üí Examples ‚Üí Advanced
    • ‚úÖ 15-Minute Tutorial - Quick-start guide for new users
    • ‚úÖ Cross-References - Related commands and documentation links
    • ‚úÖ Visual Hierarchy - Box borders, color coding, clear sections

    Quick Start¶

    View Main Help¶

    $ teach --help
# or
$ teach help
# or
$ teach -h

    Shows complete dispatcher overview with all command categories.

    View Command-Specific Help¶

    $ teach lecture --help       # Lecture generation help
$ teach exam -h              # Exam creation help
$ teach validate help        # Validation help

    All three formats work: --help, -h, help

    Follow the Tutorial¶

    ADHD-Friendly Design Principles¶

    The help system follows ADHD-friendly design:

    1. Scannable Structure¶

    ‚úÖ Box borders - Clear visual boundaries ‚úÖ Bold section headers - Easy to scan ‚úÖ Categorized options - Grouped by purpose ‚úÖ Short examples first - Quick wins at the top

    2. Progressive Complexity¶

    ‚úÖ Quick Start first - 3 examples you can copy-paste ‚úÖ Options next - Categorized, not alphabetical ‚úÖ Examples with explanations - What happens when you run this ‚úÖ Tips last - Advanced users can keep reading

    3. Consistent Patterns¶

    ‚úÖ Same structure everywhere - Learn once, use everywhere ‚úÖ Same color coding - Visual consistency ‚úÖ Same terminology - No synonyms for same concept

    4. Low Cognitive Load¶

    ‚úÖ 3 Quick Start examples - Not 1 (too rigid), not 5 (too many) ‚úÖ Grouped flags - By purpose, not alphabet ‚úÖ Inline comments - Explain why, not just what ‚úÖ Clear next steps - "SEE ALSO" shows related commands

    Help Routing¶

    How Help is Invoked¶

    Commands support three help formats:

    teach lecture --help     # Long flag
teach lecture -h         # Short flag
teach lecture help       # Positional argument

    Routing Implementation¶

    lecture|lec)
    case "$1" in
        --help|-h|help)
            _teach_lecture_help
            return 0
            ;;
        *)
            _teach_scholar_wrapper "lecture" "$@"
            ;;
    esac
    ;;

    Pattern: 1. Check if first argument is help flag 2. Call help function and exit (return 0) 3. Otherwise pass to command implementation
