    Teaching Commands - Detailed Guide¶

    Version: 1.2 Status: Production Ready Last Updated: 2026-01-18

    Overview¶

    This guide provides comprehensive, step-by-step explanations of each teaching command, what it does, why you'd use it, and real-world workflows.

    Teaching Workflow Architecture¶

    The teaching workflow follows a draft ‚Üí production pattern that keeps your work safe while giving students a stable experience:

    flowchart TB
    subgraph Setup["üéì One-Time Setup"]
        init["teach init"]
    end

    subgraph Daily["üìù Daily Workflow"]
        work["work course-name"]
        edit["Edit materials"]
        commit["git commit"]
        deploy["teach deploy"]
    end

    subgraph Monitor["üëÄ Monitoring"]
        status["teach status"]
        week["teach week"]
    end

    subgraph Semester["üì¶ End of Semester"]
        archive["teach archive"]
        config["teach config"]
    end

    init --> work
    work --> edit
    edit --> commit
    commit --> deploy
    deploy --> status

    status -.-> week
    week -.-> edit

    deploy --> archive
    archive --> config
    config --> work

    style deploy fill:#2196F3,color:#fff
    style archive fill:#FF9800,color:#fff

    Branch Strategy¶

    gitGraph
    commit id: "Initial"
    branch draft
    checkout draft

    merge draft id: "Deploy v1" tag: "live"

    checkout draft

    Key Principle: You always work on draft. Students always see production. The teach deploy command safely moves changes between them.

    Command Breakdown¶

    1. teach init - Initialize a New Course¶

    What it does: Sets up a complete teaching workflow for a course repository. This one-time setup creates all the infrastructure needed for course website management.

    Process Flowchart¶

    flowchart TD
    Start([teach init "Course"]) --> CheckGit{Git repo?}
    CheckGit -->|No| InitGit[git init]
    CheckGit -->|Yes| CheckFlags{Check flags}
    InitGit --> CheckFlags

    CheckFlags -->|--dry-run| DryRun[Show plan only]
    CheckFlags -->|-y/--yes| SafeDefaults[Use safe defaults]
    CheckFlags -->|none| Interactive[Interactive prompts]

    DryRun --> End([Done])

    SafeDefaults --> CreateConfig
    Interactive --> PromptSemester[Prompt: Semester?]
    PromptSemester --> PromptDates[Prompt: Start/End dates?]
    PromptDates --> PromptBreaks[Prompt: Include breaks?]
    PromptBreaks --> PromptPush[Prompt: Push to GitHub?]
    PromptPush --> CreateConfig

    CreateConfig[Create .flow/teach-config.yml] --> CreateScripts[Create scripts/]
    CreateScripts --> CreateWorkflows[Create .github/workflows/]
    CreateWorkflows --> SetupBranches[Setup draft/production branches]
    SetupBranches --> GitCommit[Create initial commit]
    GitCommit --> CheckPush{Push to GitHub?}

    CheckPush -->|Yes| GitPush[git push -u origin draft production]
    CheckPush -->|No| ShowNext[Show next steps]
    GitPush --> ShowNext
    ShowNext --> End

    style Start fill:#4CAF50,color:#fff
    style CreateConfig fill:#2196F3,color:#fff
    style End fill:#9E9E9E,color:#fff

    Decision Tree¶

    Flag Behavior Use Case
    (none) Interactive prompts First-time setup, custom config
    -y / --yes Safe defaults, no prompts Quick setup, CI/CD
    --dry-run Preview only, no changes See what would happen

    Syntax:

    teach init "Course Name"              # Interactive mode
teach init -y "Course Name"           # Non-interactive (safe defaults)
teach init --dry-run "Course Name"    # Preview without changes

    When to use: - First time setting up a course repository - Migrating an existing course to flow-cli - Setting up a new semester variant of a course

    What happens step-by-step:

    1. Configuration Creation
    2. Creates .flow/teach-config.yml configuration file
    3. Stores course name, semester, schedule, and branch names

    4. Example:

      course:
  name: "STAT 545"
  semester: "spring"
  year: 2026

branches:
  draft: "draft"           # Where you make edits
  production: "production" # What students see

semester_info:
  start_date: "2026-01-13"
  end_date: "2026-05-08"

    5. Branch Setup

    6. Creates draft and production branches
    7. draft: Your working branch (safe to edit)
    8. production: Student-facing branch (auto-deployed)

    9. Sets draft as default branch

    10. Script Installation

    11. Creates scripts/quick-deploy.sh - One-command deployment
    12. Creates scripts/semester-archive.sh - End-of-semester archival

    13. Both scripts are executable and ready to use

    14. GitHub Actions Setup

    15. Creates .github/workflows/ if needed
    16. Configures automatic deployment to GitHub Pages
    17. Triggers on production branch updates

    18. Students see updates automatically

    19. Git Commit

    20. Stages all new files
    21. Creates initial commit with setup
    22. Ready for pushing to GitHub

    Options:

    -y or --yes (Non-interactive mode): - Accepts safe defaults without prompting - Great for automated setup or when you know what you want - Defaults: - ‚úÖ Keep git history (in-place conversion) - ‚úÖ Auto-exclude build artifacts (renv, _build, etc) - ‚úÖ Use sensible semester dates - ‚úÖ Skip GitHub push (do manually)

    --dry-run: - Shows what would happen without making changes - Preview the setup plan - Useful to see what files will be created

    Interactive prompts: - Course name (already provided) - Semester and year - Branch names (usually draft and production) - Semester start/end dates - Include breaks? (Spring Break, etc) - Push to GitHub now? (optional)

    Example:

    # Initialize a new course - interactive
cd ~/projects/teaching/stat-545
teach init "STAT 545"

# Output:

# ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ‚îÅ
#
# Course Name: STAT 545
# Semester: Spring
# Year: 2026
#
# ‚úì Created .flow/teach-config.yml
# ‚úì Created scripts/quick-deploy.sh

# ‚úì Configured GitHub Actions
# ‚úì Initial commit created
#
# Next: push to GitHub
# $ git push -u origin draft production

    Non-interactive (safe defaults):

    teach init -y "STAT 440"
# Uses all defaults, completes in seconds

    2. teach deploy - Deploy Changes to Production¶

    What it does: Moves your changes from the draft branch (where you edit) to the production branch (what students see) via a PR workflow. Safely handles pre-flight checks, branch switching, and triggers GitHub Pages deployment.

    Process Flowchart¶

    flowchart TD
    Start([teach deploy]) --> CheckGit{In git repo?}
    CheckGit -->|No| ErrorGit[‚ùå Not a git repository]
    CheckGit -->|Yes| CheckConfig{Config exists?}

    CheckConfig -->|No| ErrorConfig[‚ùå .flow/teach-config.yml not found]
    CheckConfig -->|Yes| LoadConfig[Load branch configuration]

    ErrorGit --> End([Exit])
    ErrorConfig --> HintInit[üí° Run 'teach init' first]
    HintInit --> End

    LoadConfig --> CheckBranch{On draft branch?}
    CheckBranch -->|No| PromptSwitch{Switch to draft?}
    PromptSwitch -->|Yes| SwitchBranch[git checkout draft]
    PromptSwitch -->|No| Cancel[Cancelled]
    Cancel --> End
    SwitchBranch --> CheckClean

    CheckBranch -->|Yes| CheckClean{Clean working tree?}
    CheckClean -->|No| ErrorDirty[‚ùå Uncommitted changes]
    ErrorDirty --> HintCommit[üí° Commit or stash first]
    HintCommit --> End

    CheckClean -->|Yes| CheckRemote{Remote up-to-date?}
    CheckRemote -->|No| WarnRemote[‚ö†Ô∏è Remote has changes]
    WarnRemote --> PromptRebase{Rebase first?}
    PromptRebase -->|Yes| Rebase[git rebase origin/draft]
    PromptRebase -->|No| Continue
    Rebase --> Continue

    CheckRemote -->|Yes| Continue[Continue deployment]

    Continue --> CheckProd{Production has updates?}
    CheckProd -->|Yes| PromptMerge{Rebase onto production?}
    PromptMerge -->|Yes| RebaseProd[git rebase origin/production]
    PromptMerge -->|No| SkipRebase[Continue anyway]
    RebaseProd --> CreatePR
    SkipRebase --> CreatePR

    CheckProd -->|No| CreatePR{--direct-push?}

    CreatePR -->|Yes| DirectMerge[Merge draft ‚Üí production]
    CreatePR -->|No| PRWorkflow[Create PR: draft ‚Üí production]

    DirectMerge --> Push[git push origin production]
    PRWorkflow --> Push

    Push --> Trigger[üöÄ GitHub Actions triggered]
    Trigger --> Deploy[üì¶ Deploy to GitHub Pages]
    Deploy --> Success[‚úÖ Deployment complete]
    Success --> ShowURL[üåê Show live URL]
    ShowURL --> End

    style Start fill:#2196F3,color:#fff
    style Success fill:#4CAF50,color:#fff
    style ErrorGit fill:#f44336,color:#fff
    style ErrorConfig fill:#f44336,color:#fff
    style ErrorDirty fill:#f44336,color:#fff
    style End fill:#9E9E9E,color:#fff

    Pre-flight Checks Summary¶

    Check Pass Fail Recovery
    Git repository ‚úì Continue ‚ùå Exit git init
    Config file ‚úì Continue ‚ùå Exit teach init
    On draft branch ‚úì Continue ‚ö†Ô∏è Prompt Auto-switch or manual
    Clean working tree ‚úì Continue ‚ùå Exit Commit or stash
    Remote synced ‚úì Continue ‚ö†Ô∏è Warn Optional rebase
    Production updated ‚úì Continue ‚ö†Ô∏è Prompt Optional rebase

    Syntax:

    teach deploy              # Standard PR workflow
teach deploy --direct-push # Bypass PR (advanced users only)

    Requirements: - Must be in a git repository - Config file must exist at .flow/teach-config.yml - Must be on the draft branch (or will prompt to switch)

    When to use: - After making changes to lectures, assignments, solutions - Publishing new course materials - Fixing typos on the live site - Multiple times per day (very fast!)

    What happens step-by-step:

    1. Pre-flight Checks (v5.13.0+)
    2. Verifies .flow/teach-config.yml exists
    3. Reads branch configuration from config
    4. Verifies you're on draft branch (offers to switch if not)
    5. Checks for uncommitted changes
    6. Checks if remote is up-to-date

    7. Checks if production has new commits (offers rebase)

    8. Branch Configuration The deploy command reads branch names from .flow/teach-config.yml:

    # Preferred format (v5.11.0+)
branches:
  draft: draft
  production: main

# Legacy format (still supported)
git:
  draft_branch: draft
  production_branch: main
    1. Branch Merge
    draft (your edits)
   ‚Üì
production (students see this)
   ‚Üì
GitHub (via PR or direct push)
   ‚Üì
GitHub Pages (automatic deployment)
   ‚Üì
Live website
    1. Git Operations
    2. Creates PR from draft ‚Üí production (default)
    3. Or direct merge with --direct-push
    4. Returns to draft branch

    5. Shows merge summary

    6. Deployment Trigger

    7. Pushes to GitHub
    8. GitHub Actions workflow starts automatically
    9. Site rebuilds and deploys to GitHub Pages

    10. Usually takes 1-2 minutes

    11. Completion Status

    12. Shows successful deployment message
    13. URL to live site
    14. Suggestion to verify changes

    Example: Daily Typo Fix

    # You notice a typo on the live site
cd ~/projects/teaching/stat-545
work stat-545          # Starts work session on draft branch

# Make edits...
# Fix: "Lineare regression" ‚Üí "Linear regression" in lectures/week05.qmd

git add lectures/week05.qmd
git commit -m "fix: correct spelling in week 5"

# Deploy immediately
teach deploy

    Error scenarios:

    On production branch warning:

    ‚ùå Must be on draft branch
Current branch: production
Run: git checkout draft

    Uncommitted changes:

    ‚ùå Uncommitted changes found
Commit or stash changes first:
$ git add .
$ git commit -m "your message"

    Merge conflicts: - Automatically resolves most conflicts - If manual intervention needed, shows instructions - Always safe to attempt (reverts on conflict)

    3. teach status - View Course Status¶

    What it does: Shows a dashboard of your current course including semester info, week number, branch status, and safety indicators.

    Process Flowchart¶

    flowchart TD
    Start([teach status]) --> CheckConfig{Config exists?}
    CheckConfig -->|No| ErrorConfig[‚ùå No teaching config found]
    CheckConfig -->|Yes| LoadConfig[Load .flow/teach-config.yml]

    ErrorConfig --> End([Exit])

    LoadConfig --> GetCourse[Read course info]
    GetCourse --> GetSemester[Calculate semester dates]
    GetSemester --> GetWeek[Calculate current week]
    GetWeek --> GetBranch[Check current git branch]
    GetBranch --> GetDirty[Check uncommitted changes]
    GetDirty --> GetDeploy[Check last deployment]

    GetDeploy --> RenderDashboard[Render Dashboard]

    subgraph Dashboard["üìä Dashboard Output"]

        WeekInfo["üìÖ Week X of Y"]
        BranchStatus["üîÄ Branch status"]
        SafetyIndicator["üö¶ Safety indicator"]
        DeployStatus["üöÄ Deploy status"]
    end

    RenderDashboard --> CourseInfo
    CourseInfo --> WeekInfo
    WeekInfo --> BranchStatus
    BranchStatus --> SafetyIndicator
    SafetyIndicator --> DeployStatus
    DeployStatus --> End

    5. teach archive - Archive a Semester¶

    What it does: Creates a permanent snapshot of your semester at its completion. Useful for: - Preserving final state of course - Starting fresh for next semester - Comparing changes between years - Regulatory/institutional requirements

    Process Flowchart¶

    flowchart TD
    Start([teach archive]) --> LoadConfig[Load .flow/teach-config.yml]
    LoadConfig --> GetInfo[Get course & semester info]
    GetInfo --> GenerateTag[Generate tag name]

    GenerateTag --> ShowPlan["Show archive plan:<br/>‚Ä¢ Tag name<br/>‚Ä¢ Semester dates<br/>‚Ä¢ File count"]

    ShowPlan --> Confirm{Confirm archive?}
    Confirm -->|No| Cancel[Cancelled]
    Cancel --> End([Exit])

    Confirm -->|Yes| CheckDirty{Clean working tree?}
    CheckDirty -->|No| WarnDirty[‚ö†Ô∏è Uncommitted changes]
    WarnDirty --> PromptCommit{Commit first?}
    PromptCommit -->|Yes| AutoCommit[git commit -m 'final semester state']
    PromptCommit -->|No| Cancel
    AutoCommit --> CreateTag

    CheckDirty -->|Yes| CreateTag[Create annotated git tag]

    CreateTag --> PushTag{Push tag to remote?}
    PushTag -->|Yes| GitPush[git push origin --tags]

    GitPush --> ShowNext
    LocalOnly --> ShowNext[Show next steps]

    ShowNext --> NextSteps["üí° Next steps:<br/>1. Update teach-config.yml<br/>2. Change semester/year<br/>3. Commit & deploy"]

    Troubleshooting¶

    ".flow/teach-config.yml not found" error¶

    Problem: The teaching configuration file doesn't exist

    Solution:

    # Initialize teaching workflow for this course
teach init "Course Name"

# Or with non-interactive mode
teach init -y "Course Name"

    What this creates: - .flow/teach-config.yml - Configuration file - scripts/quick-deploy.sh - Deployment script - Branch structure (draft/production)

    "Not on draft branch" error¶

    Problem: You tried to deploy but you're on the wrong branch

    Solution:

    # Option 1: Let teach deploy switch for you
teach deploy
# ‚Üí Will prompt: "Switch to draft branch? [Y/n]"
# ‚Üí Type 'y' to switch automatically

# Option 2: Switch manually
git checkout draft
teach deploy

    Prevention: Work sessions automatically warn if on production

    "Uncommitted changes" warning¶

    Problem: You made edits but forgot to commit

    Solution:

    # Check what changed
git status

# Stage changes
git add .

# Commit with message
git commit -m "fix: update lecture notes"

# Now deploy
teach deploy

    Deploy seems slow¶

    Problem: Deployment taking > 3 minutes

    Check:

    # View GitHub Actions status
gh run list

# Or visit GitHub Actions tab:
# github.com/your-org/repo-name/actions

    Usually caused by: - Large file changes (PDFs, images) - First-time GitHub Pages setup - GitHub service delays (rare)
