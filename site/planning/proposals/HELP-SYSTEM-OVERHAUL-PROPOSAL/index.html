    Pain Points Identified¶

    1. No Visual Hierarchy - All text looks the same
    2. No Examples - Users don't see actual usage
    3. No Context - Can't see "related commands" or "typical workflow"
    4. Static Only - No interactive elements
    5. Overwhelming - Long lists for functions like r and cc
    6. No Quick Reference - Can't get just the essentials
    7. No Colors - Hard to scan quickly (ADHD issue)
    8. No Search - Can't find specific action quickly

    Core Principles¶

    1. Cognitive Load Reduction
    2. Quick scan in <3 seconds
    3. Essential info first, details on demand

    4. Visual cues (colors, icons, hierarchy)

    5. Multiple Access Patterns

    6. Quick mode: Show just essentials
    7. Full mode: Complete reference
    8. Example mode: See it in action

    9. Search mode: Find specific action

    10. Progressive Disclosure

    11. Start with most common commands
    12. Expand to advanced features

    13. Layer complexity

    14. Visual Differentiation

    15. Colors for categories
    16. Icons for command types
    17. Highlighting for emphasis
