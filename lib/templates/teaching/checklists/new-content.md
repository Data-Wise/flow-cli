# New Content Checklist

Use this checklist when creating new lectures, labs, or assignments.

## Planning

- [ ] Topic identified and scoped
- [ ] Week/module placement determined
- [ ] Prerequisites identified
- [ ] Learning objectives drafted (3-5)
- [ ] Time estimate made

## Template Selection

- [ ] Appropriate template chosen:
  - [ ] `teach templates new lecture` - Lecture notes
  - [ ] `teach templates new slides` - Presentation
  - [ ] `teach templates new lab` - R lab exercise
  - [ ] `teach templates new assignment` - Homework

## Content Development

- [ ] Introduction written
- [ ] Main content structured with clear sections
- [ ] Examples included (at least 2)
- [ ] Code chunks labeled and documented
- [ ] Visualizations created (if applicable)
- [ ] Practice problems/exercises added
- [ ] Summary section completed

## Concept Mapping (for lectures)

- [ ] Concept IDs defined
- [ ] Prerequisites linked
- [ ] `teach analyze` run to validate

## Review

- [ ] Content reviewed for accuracy
- [ ] Difficulty level appropriate
- [ ] Timing estimated and reasonable
- [ ] Peer review (if available)

## Integration

- [ ] File placed in correct directory
- [ ] `_quarto.yml` updated (if needed)
- [ ] Cross-references checked
- [ ] Index page updated (if applicable)

## Final Validation

- [ ] `teach validate` passes
- [ ] Local preview checked
- [ ] Git commit made
