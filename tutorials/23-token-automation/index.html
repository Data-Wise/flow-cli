    Tutorial 23: Token Automation¶

    Smart token management with caching and isolated checks

    Time: ~15 minutes | Level: Intermediate | v5.17.0

    What You'll Learn¶

    By the end of this tutorial, you'll be able to:

    • ‚úÖ Use doctor --dot for fast, isolated token checks
    • ‚úÖ Understand the smart caching system (5-min TTL)
    • ‚úÖ Fix token issues with the interactive menu
    • ‚úÖ Control output verbosity for different use cases
    • ‚úÖ Integrate token validation into workflows

    Prerequisites¶

    Before starting:

    • ‚úì Flow-CLI installed (flow doctor works)
    • ‚úì GitHub CLI configured (gh auth status)
    • ‚úì Basic understanding of doctor command

    New to flow-cli?

    Complete Tutorial 1: First Session first

    Step 1: Why Token Automation?¶

    The Problem:

    Before v5.17.0, checking GitHub token health required a full doctor run: - Takes 60+ seconds (checks shell, tools, integrations, dotfiles, tokens) - Slow for quick token validation - Not practical for CI/CD automation - No caching (repeated checks hit GitHub API)

    The Solution:

    Token automation provides: - ‚ö° 20x faster - Isolated checks in < 3s (vs 60s) - üíæ Smart caching - 5-minute TTL, 85% hit rate - üéØ Focused workflows - Check only what matters - üîä Verbosity control - Quiet for scripts, verbose for debugging

    Step 4: Cache Demonstration¶

    Let's understand the caching system:

    sequenceDiagram
    participant User
    participant Doctor
    participant Cache
    participant GitHub

    User->>Doctor: doctor --dot
    Doctor->>Cache: Check cache (< 5 min old?)

    alt Cache HIT
        Cache-->>Doctor: Return cached result
        Doctor-->>User: ‚úì Valid (cached, 0.05s)
    else Cache MISS
        Doctor->>GitHub: Validate token
        GitHub-->>Doctor: Token valid, expires 45d
        Doctor->>Cache: Store result (TTL: 5 min)
        Doctor-->>User: ‚úì Valid (fresh, 2.3s)
    end

    Cache Lifecycle¶

    Scenario Cache Status Speed API Call
    First check MISS ~2-3s Yes
    Within 5 min HIT ~50-80ms No
    After 5 min MISS ~2-3s Yes
    After dot token rotate Invalidated ~2-3s Yes

    Performance Impact: - 85% cache hit rate in typical workflows - 80% API call reduction (respects rate limits) - Sub-10ms cache lookups (5-8ms average)

    Step 8: CI/CD Automation¶

    Use quiet mode in automated environments:

    GitHub Actions¶

    # .github/workflows/ci.yml
name: CI
on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check GitHub token
        run: doctor --dot --quiet

      - name: Run tests
        if: success()
        run: ./tests/run-all.sh

    Cron Job¶

    # Check token health daily
0 9 * * * /usr/local/bin/doctor --dot --quiet || echo "Token check failed!" | mail -s "flow-cli alert" you@example.com

    Shell Script¶

    #!/bin/bash
# deploy.sh - Automated deployment script

# Validate token before deployment

    echo "‚ùå GitHub token invalid. Run: doctor --fix-token"
