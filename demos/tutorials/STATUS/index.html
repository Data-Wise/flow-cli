    Recording Method Change¶

    Previous: VHS tapes with simulated output ‚ùå Current: asciinema recordings of real Claude Code sessions ‚úÖ

    Why the change? - ‚úÖ Real command output (authentic Scholar Enhancement demos) - ‚úÖ Natural timing (no need to simulate pauses) - ‚úÖ Easy to maintain (just re-record if features change) - ‚úÖ Better compression (smaller file sizes)

    Quick Start¶

    Record Demo 1 (Help System)¶

    cd ~/projects/dev-tools/flow-cli/docs/demos/tutorials

# Start recording
asciinema rec scholar-01-help.cast

# In the recording terminal:
claude
teach slides --help
teach quiz --help
teach lecture --help
exit

# Recording stops automatically
# Convert to GIF
agg \
  --cols 100 \
  --rows 30 \
  --font-size 16 \
  --theme dracula \
  --fps-cap 10 \
  scholar-01-help.cast scholar-01-help.gif

# Optimize
gifsicle -O3 --colors 128 --lossy=80 \
  scholar-01-help.gif -o scholar-01-help.gif

# Check size
ls -lh scholar-01-help.gif

    Recording Guidelines¶

    Terminal Setup¶

    Before recording: - Clear terminal: clear - Set consistent size: Terminal ‚Üí Window ‚Üí 100x30 - Use clean shell (no custom prompt distractions)

    During Recording¶

    Do: - ‚úÖ Wait 2-3 seconds before starting commands - ‚úÖ Let commands complete fully - ‚úÖ Pause 2 seconds after output - ‚úÖ Exit cleanly

    Don't: - ‚ùå Rush through commands - ‚ùå Make typos (can't edit easily) - ‚ùå Include sensitive information - ‚ùå Show errors (unless demonstrating error handling)

    Conversion Settings¶

    Recommended: 

    agg \
  --cols 100 \          # Consistent width
  --rows 30 \           # Consistent height
  --font-size 16 \      # Readable on docs site
  --theme dracula \     # Good contrast
  --fps-cap 10 \            # Smooth playback
  input.cast output.gif

    Alternative themes: - monokai - Popular dark theme - solarized-dark - Lower contrast - nord - Cool blue tones

    FPS trade-offs: - fps 5 - Smaller files, slightly choppy - fps 10 - Good balance (recommended) - fps 15 - Very smooth, larger files

    Demo Target Acceptable Too Large

    Simple output < 300 KB 300-400 KB > 400 KB

    Troubleshooting¶

    "command not found: teach"¶

    Solution: Record in a project directory with Scholar Enhancement configured:

    cd ~/projects/teaching/stat-440  # Or any teaching project
asciinema rec scholar-XX.cast

    GIF too large (> 500 KB)¶

    Solution 1: Increase compression 

    gifsicle -O3 --colors 64 --lossy=90 \
  input.gif -o output.gif

    Solution 2: Reduce FPS 

    agg --fps-cap 5 input.cast output.gif

    Solution 3: Trim duration 

    # Edit .cast file (it's JSON)
# Remove frames at beginning/end

    Poor quality after optimization¶

    Solution: Reduce lossy compression 

    gifsicle -O3 --colors 256 --lossy=60 \
  input.gif -o output.gif

    When to Re-record¶

    Re-record demos if:

    1. ‚úÖ Feature changes - Commands or output format changes
    2. ‚úÖ Errors in demo - Typos or incorrect steps
    3. ‚úÖ Quality issues - Text unreadable or poor contrast
    4. ‚ùå Minor tweaks - Don't re-record for tiny changes
