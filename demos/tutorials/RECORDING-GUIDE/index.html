    Scholar Enhancement GIF Recording Guide¶

    Updated: 2026-01-17 Method: asciinema + agg (real Claude Code sessions)

    Prerequisites¶

    ‚úÖ Installed: - asciinema (terminal recorder) - agg (asciinema gif generator) - gifsicle (GIF optimizer)

    brew install asciinema agg gifsicle

    Recording Workflow¶

    1. Start Recording¶

    cd ~/projects/dev-tools/flow-cli/docs/demos/tutorials
asciinema rec scholar-01-help.cast

    2. Run Commands in Claude Code¶

    In the recording terminal:

    # Start Claude Code
claude

# Now inside Claude Code, run Scholar commands:
teach slides --help
# Wait for output, then exit Claude Code
exit

    3. Stop Recording¶

    # Recording automatically stops when you exit
# Or press Ctrl+D

    4. Convert to GIF¶

    # Basic conversion
agg scholar-01-help.cast scholar-01-help.gif

# With customization (recommended)
agg \
  --cols 100 \
  --rows 30 \
  --font-size 16 \
  --theme dracula \
  --fps-cap 10 \
  scholar-01-help.cast scholar-01-help.gif

    5. Optimize GIF¶

    # Optimize with gifsicle
gifsicle -O3 --colors 128 --lossy=80 \
  scholar-01-help.gif -o scholar-01-help.gif

# Check size
ls -lh scholar-01-help.gif

    Batch Conversion Script¶

    Save as convert-all.sh:

    #!/bin/bash

# Convert all .cast files to GIFs
for cast in scholar-*.cast; do
    gif="${cast%.cast}.gif"
    echo "Converting $cast ‚Üí $gif"

    # Convert with agg
    agg \
      --cols 100 \
      --rows 30 \
      --font-size 16 \
      --theme dracula \
      --fps-cap 10 \
      "$cast" "$gif"

    # Optimize with gifsicle
    gifsicle -O3 --colors 128 --lossy=80 \
      "$gif" -o "$gif"

    # Show size
    ls -lh "$gif"
done

echo "‚úÖ All GIFs generated"

    Usage:

    chmod +x convert-all.sh
./convert-all.sh

    Tips¶

    Terminal Size¶

    • Use --cols 100 --rows 30 for consistent dimensions
    • Matches documentation site width

    Theme¶

    • dracula theme has good contrast
    • Alternatives: monokai, solarized-dark

    Frame Rate¶

    • --fps-cap 10 = smooth but large files
    • --fps-cap 5 = smaller files, slightly choppy
    • --fps-cap 15 = very smooth but very large

    Optimization¶

    • --lossy=80 reduces size by ~40%
    • --colors 128 vs --colors 256 saves ~20%
    • -O3 is maximum optimization

    File Sizes¶

    Target: 100-500 KB per GIF - Help screens: ~100-200 KB - Command output: ~200-400 KB - Interactive sessions: ~300-500 KB

    Quality Check¶

    Before committing GIFs:

    # Check sizes
ls -lh scholar-*.gif

# View in browser
open scholar-01-help.gif

# Verify quality
gifsicle --info scholar-01-help.gif

    Why asciinema + agg?¶

    Advantages over VHS: 1. ‚úÖ Real output - No simulation needed 2. ‚úÖ Timing preserved - Natural command execution 3. ‚úÖ Easy editing - Can edit .cast JSON file 4. ‚úÖ Small files - Better compression 5. ‚úÖ Theme support - Dracula, Monokai, etc.

    VHS Issues: - ‚ùå Requires simulating output (fragile) - ‚ùå Timing feels artificial - ‚ùå Hard to maintain (rewrite tapes for changes)
