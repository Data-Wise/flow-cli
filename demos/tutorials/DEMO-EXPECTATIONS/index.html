    Scholar Enhancement Demo - Expected Outputs¶

    Purpose: This guide explains what you should see when running each demo command.

    Demo 1: Help System¶

    Command¶

    teach slides --help
teach quiz --help
teach lecture --help

    What You'll See¶

    For each --help command:

    1. Command Description
    2. Usage pattern with topic and options

    3. Brief explanation of what the command generates

    4. Universal Flags (v5.13.0+)

    5. Topic selection: --topic, --week
    6. Content style presets: conceptual, computational, rigorous, applied
    7. Content customization: ~12 flags (explanation, definitions, proof, math, examples, code, etc.)

    8. Workflow modes: interactive, revise, context

    9. Command-Specific Options

    10. For slides: theme, format, from-lecture
    11. For quiz: number of questions, duration, difficulty

    12. For lecture: outline-only, detailed, format

    13. Examples Section

    14. Quick start example
    15. Style preset usage
    16. Advanced customization

    Expected Behavior: - Each help screen is ~40-60 lines - Color-coded sections (headers in blue/purple) - Organized into logical groups - Examples at the end

    # Navigate to course
cd ~/projects/teaching/scholar-demo-course

# Ensure flow-cli is loaded
source /Users/dt/.git-worktrees/flow-cli/feature/teaching-flags/flow.plugin.zsh

# Run any teach command
teach slides "Data Visualization" --style computational
teach lecture --lesson content/lesson-plans/week03.yml
teach quiz "Probability" --style conceptual --questions 10

    Course Configuration¶

    The teach-config.yml provides: - Course metadata (STAT 101, Spring 2026) - Semester dates and weeks - Scholar defaults (style, format, tone) - Grading breakdown - Topics list

    When you run teach commands in this directory, Scholar uses this configuration automatically.

    Troubleshooting¶

    Command Not Found: teach¶

    Problem: teach: command not found

    Solution:

    # Load flow-cli first
cd /Users/dt/.git-worktrees/flow-cli/feature/teaching-flags
source flow.plugin.zsh

    Not a Teaching Project¶

    Problem: "Not in a teaching project directory"

    Solution:

    # Navigate to teaching course first
cd ~/projects/teaching/scholar-demo-course

# Or run teach init in a directory
teach init "Course Name"

    Scholar Not Available¶

    Problem: "Scholar commands require Claude Code CLI"

    Solution: The demos use simulated output, but real commands require: 1. Claude Code CLI installed 2. Scholar plugin loaded 3. Teaching project configured
